L3
Memoria · RAG & Conocimiento
Contexto externo
Un LLM solo conoce su entrenamiento. RAG (Retrieval-Augmented Generation) le da acceso a tus documentos en tiempo real: troceas → vectorizas → guardas → recuperas al preguntar.

NotebookLM hace esto automáticamente. Open WebUI RAG también, con tus modelos locales. Para RAG en tus apps Python usarías ChromaDB + LlamaIndex.

Conceptos clave
¿Qué es un embedding?
Representación numérica del texto como vector. Textos con significado similar quedan "cerca" geométricamente. Permite buscar por semántica, no solo palabras exactas.
NotebookLM vs RAG propio
NotebookLM: sube PDFs, funciona solo, sin código.
RAG propio: integrado en tu app Python, control total sobre qué documentos y cómo se recuperan.
Claude Projects como RAG ligero
Sube documentos a un Project y quedan disponibles en todas las conversaciones. Sin código, sin infraestructura.

Guía formativa · El stack de abajo a arriba
El diagrama va de abajo a arriba — como los pisos de un edificio: los cimientos sostienen todo lo demás. Cada capa depende de las que tiene debajo, pero no necesita las de arriba. L1 funciona sola. L6 necesita a todas.
L1
Modelo · LLM Core
Núcleo generativo
Los cimientos
Es el único nivel que genera inteligencia. Todo lo demás es fontanería alrededor. Sin él, nada funciona. Aquí viven Claude, GPT-4o, y tu Llama local en Ollama. Son "cerebros" sin manos: saben razonar pero no pueden hacer nada por sí solos.
L2
Interfaz · Chat & Studios
Punto de acceso
La ventana
Es simplemente cómo llegas al modelo. Claude.ai, Open WebUI, el Playground de OpenAI... son interfaces que envuelven al LLM y lo hacen usable. Sin L1 debajo, no existe. Es la capa más "visible" pero la menos inteligente.
L3
Memoria · RAG & Conocimiento
Contexto externo
La biblioteca
El LLM no recuerda nada entre conversaciones y solo conoce lo que aprendió en su entrenamiento. Esta capa le da acceso a tus documentos en tiempo real. NotebookLM es L3: subes un PDF, lo trocea, lo vectoriza, y cuando preguntas recupera los fragmentos relevantes antes de responder. Sin L1, no puede responder. Sin L2, no puedes preguntar.
L4
Herramientas · Tools & Conexiones
Extensión del modelo
Las manos
Aquí el modelo deja de solo hablar y empieza a actuar: buscar en internet, leer un email, crear un evento, llamar a una API. MCP es el protocolo que lo hace posible. n8n orquesta esos flujos. Esta capa requiere L1 (para decidir qué herramienta usar) y L2 (para recibir la instrucción).
L5
Agentes · IA Autónoma
Coding · Builders · Browser
El autónomo
Combina todo lo anterior en bucles: el modelo planifica → usa herramientas (L4) → observa el resultado → ajusta → repite. No espera instrucción por instrucción. Antigravity hace esto con tu código: recibe un objetivo, lee el repo entero, ejecuta cambios en múltiples archivos. Copilot es más simple: sugiere mientras escribes, tú decides.
L6
Aplicación · Tus Proyectos
Capa de salida
Tu negocio
Donde todo el stack se convierte en valor concreto para ti. Tus tres apps Python, la web de espacioGijón, el portfolio. Esta capa consume las inferiores según necesite: llama a la API de Claude (L1 vía L4), puede usar RAG (L3), puede lanzar agentes (L5).

⚡ La regla de oro

Cada capa depende de las que tiene debajo, pero no necesita las de arriba. L1 funciona sola — es el único nivel que genera inteligencia. L6 necesita a todas — es donde el stack se convierte en valor real. Puedes usar solo L1+L2 para el 80% de los casos de uso diarios.

Atajos en VS Code
Tab
Aceptar sugerencia de Copilot. El atajo más usado. Acepta la sugerencia completa actual.
Esc
Rechazar sugerencia actual.
Alt + ]
Ver siguiente sugerencia alternativa (si hay varias).
Alt + [
Ver sugerencia anterior.
Ctrl + Enter
Abrir panel con todas las sugerencias. Ver múltiples opciones a la vez.
Ctrl + I
Abrir Copilot Chat inline. Puedes preguntar sobre el código seleccionado directamente.
Ctrl + Shift + I
Abrir panel de Copilot Chat lateral. Para conversaciones largas o exploración.
Comandos de Copilot Chat
/explain
Explicar el código seleccionado o el archivo actual.
/fix
Proponer arreglo para un error o problema en el código.
/tests
Generar tests unitarios para el código seleccionado.
/doc
Generar documentación (docstring) para función o clase seleccionada.
@workspace
Dar contexto de todo el proyecto al hacer la pregunta.
@terminal
Dar contexto del terminal (últimos comandos/errores) para diagnosticar problemas.
#file:nombre.py
Incluir un archivo específico en el contexto de la pregunta.
Mejores prácticas
Comentarios como prompt
Escribe un comentario describiendo lo que quieres y Copilot generará el código. Funciona en cualquier lenguaje.
Nombres descriptivos
Funciones y variables con nombres claros mejoran drásticamente la calidad de las sugerencias.
Copilot + Claude Code
Copilot para sugerencias inline (reactivo). Claude Code para cambios multi-archivo o razonamiento complejo (proactivo).
